# Security pack — distilled threat models & authorization record

> The Security/Audit pack's **settled** knowledge: the current threat models (assets, trust
> boundaries, prioritized threats, mitigations) and the authorization/scope record for any testing.
> threat-modeler and pentester-advisor curate this. An agent reads this to design against real,
> ranked risk instead of a generic checklist — and to see exactly what testing is authorized.

## Threat models
_(Per surface: assets, trust boundaries, entry points, data flows; the prioritized threats with likelihood/impact; and each threat's concrete mitigation or documented accepted risk. Filled in as surfaces are modeled.)_

## Authorization & scope record (for any active testing)
_(The owner's explicit written authorization, the agreed scope — targets in/out of bounds, allowed techniques, timing window, blast radius, rollback plan. NO active test is advised without an entry here. Default state: none on record → planning and passive/design review only.)_

## Interpreted findings
_(Real findings from authorized testing: severity, exploit chain, and the fix routed to the builders. What was covered — and, honestly, what was not.)_
