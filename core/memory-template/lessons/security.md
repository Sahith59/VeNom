# Lessons — Security

> Reusable lessons from the Security/Audit pack (threat-modeler, pentester-advisor) and any
> security-relevant finding: vulnerability classes seen in this project, false-positive patterns,
> safe test approaches, scope traps. **Knowledge compounds; rules stay fixed.** Merge duplicates and
> raise confidence; never delete history. Never edit `CHARTER.md` or any spec — rule-change ideas go
> to `../decisions/needed.md`. The security gate cross-checks these before every audit.

## Format

```
### L-<id> — <short title>   (confidence: low | medium | high)
- **Lesson:** the vulnerability class, trap, or safe approach, stated as a rule.
- **Evidence:** the audit / finding / log entry that proved it.
- **Apply when:** the situation where this lesson fires.
```
