# Lessons — Development

> Reusable lessons from the dev team: mistakes caught, patterns that worked, traps to avoid.
> **Knowledge compounds; rules stay fixed.** Add a lesson after you make or catch a mistake or find
> a reusable pattern. Merge a duplicate into the existing lesson and raise its confidence rather
> than adding a near-copy — never delete history. Never edit `CHARTER.md` or any spec; a rule-change
> idea goes to `../decisions/needed.md`, not here. The gates read these before every review, so a
> lesson here becomes an enforced check.

## Format

```
### L-<id> — <short title>   (confidence: low | medium | high)
- **Lesson:** the reusable takeaway, stated as a rule of thumb.
- **Evidence:** what proved it (ref the log entry / commit / test).
- **Apply when:** the situation where this lesson fires.
```
